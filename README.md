# Это шаблон, PyQT6

## Запуск `./.venv/bin/python3 main.py`

### 1. Общее описание
- Название: Protocol Compass
- Назначение: Клиентский лаунчер платформы

### 2. Главный экран (после авторизации)

#### 2.1. Верхняя панель (хедер)
Расположена фиксированно сверху:
- По центру:
  - Поиск проектов: Поле поиска с триграммным поиском (ищет по названию, описанию, тегам)
    - При вводе в поиск скрываются категории "Популярные", "Недавние" и т.д.
    - Показываются только результаты поиска
- Справа:
  - Баланс: Иконка + цифра (только отображение, функционал покупки не реализован)
  - Колокольчик: Иконка уведомлений (системные уведомления платформы)
  - Иконка чатов: Кликабельная (заглушка на будущее, в текущей версии может не работать)
  - Аватар пользователя: Передняя часть головы скина игрока
    - При клике открывается выпадающее меню профиль игрока

#### 2.2. Основная область контента главного экрана
Вертикальная прокрутка с проектами в виде карточек.
Категории проектов (табы/секции сверху вниз):
1. Недавние:
   - Автоматически определяются игры, в которые пользователь играл недавно
   - Сортировка по дате последнего запуска (новые слева)

2. Популярные:
   - Самые популярные проекты на текущий момент
   - Алгоритм популярности: количество активных игроков + заходов + лайков
   - Обновляется в реальном времени

3. Все игры:
   - Полный каталог проектов (без конкретного порядка в первой версии)
   - Пагинация или бесконечная прокрутка

Карточка проекта содержит:
- Превью/логотип проекта
- Название проекта
- Количество активных игроков (если онлайн)
- Рейтинг/звезды
- Теги/категории

#### 2.3. Левое боковое меню
Фиксировано слева, иконки + текст (в первой версии так):

1. Главная (домик): Возврат на главный экран
2. Игры (иконка контроллера):
   - При клике открывается страница с:
     - Поисковиком игр (отдельный от основного в хедере)
     - Блок "Случайные игры" (рандомная подборка)
     - Фильтры по категориям:
       - Мини-игры
       - Режимы
       - Выживания
       - Выживания с модами
       - [Другие категории - уточнить у Гришы]
     - Карточки игр, соответствующих выбранным фильтрам
3. Студии (иконка пока хз):
   - Сетка студий (блоки меньшего размера, чем проекты)
   - Каждая карточка студии содержит:
     - Логотип студии
     - Название студии
   - При клике на студию открывается модальное окно:
     - Крупный логотип
     - Название студии
     - Описание студии
     - Ссылки на соцсети (Discord, Telegram, VK и т.д.)
     - Список проектов студии (кликабельные)
4. Друзья (иконка людей):
   - Заглушка для будущей системы друзей и чатов
   - В текущей версии может быть неактивна
Снизу "Настройки" (шестеренка)

### 3. Модальное окно проекта
Открывается при клике на ЛЮБОЙ проект

Структура сверху вниз:
1. Шапка:
   - Крупная картинка/логотип проекта (баннер)
   - Кнопка закрытия (крестик в углу)

2. Основной контент:
   - Название проекта (крупный шрифт)
   - Информация о проекте:
     - Описание (развернутый текст)
     - Версия Minecraft
     - Размер проекта
     - Дата публикации
     - Рейтинг и количество оценок
   - Системные требования
   - Скриншоты/видео
   - Теги/категории

3. Нижняя панель (фиксировано снизу в модалке):
   - Слева: Кнопка "В избранное" (сердечко)
   - Справа: Информация об онлайне
   - По центру: Большая кнопка "Играть" (переносит в майнкрафт на сервер)

### 4. Личный кабинет в лаунчере
Открывается при клике на аватар → "Профиль"

Внутри:
- Левая колонка:
  - 3D-модель скина (статичная или с вращением)
  - Кнопка "Изменить скин"
  - Блок "Статистика"
- Правая колонка:
  - Блок "Настройки аккаунта"
  - Кнопка "Изменить данные" (email, пароль)

### 5. Технические требования

5.1. Поиск
- Триграммный поиск: Поиск по подстроке с учетом опечаток
- Индексирование: Название, описание, теги, название студии
- Фильтрация: При активном поиске скрываются все категории, показываются только результаты

5.2. Сетевые требования
- Автоматическое обновление списка проектов
- Проверка обновлений лаунчера при запуске

5.3. Игровой запуск
- Автоматическая установка нужной версии Minecraft
- Загрузка и применение необходимых модов/ресурспаков
- Запуск игры с предустановленными параметрами
- Отслеживание времени в игре (для статистики)
